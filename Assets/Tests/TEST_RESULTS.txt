ISSUE 2: Core Editor scripts inside runtime asmdef
  Severity: PRE-EXISTING — does NOT block tests, but will break player builds
  Files: 13 scripts under Assets/Core/*/Editor/ folders
    (EventChannel editors, RuntimeSet editor, SceneBootstrapper, etc.)
  Cause: GameSystems.Core.asmdef is a runtime assembly. Editor/ subfolder
         names have no special meaning inside an asmdef. These scripts use
         UnityEditor types.
  Note: This was a pre-existing issue before the test infrastructure was
        added. It only manifests in standalone builds, not in the Editor.
  Fix: Create Assets/Core/Editor/GameSystems.Core.Editor.asmdef with
       includePlatforms: ["Editor"] and references to GameSystems.Core.
       Move all 13 Core Editor scripts under it.

ISSUE 3: GameManager.OnAllObjectivesCompleted calls delegate directly
  Severity: BUG — NullReferenceException at runtime if no subscribers
  File: Assets/PaddleBall/Scripts/Managers/GameManager.cs:156
  Code:   m_GameEnded.OnEventRaised();        // <-- direct delegate call
  Should: m_GameEnded.RaiseEvent();            // <-- null-safe wrapper
  Context: EndGame() on line 124 correctly uses RaiseEvent(), but
           OnAllObjectivesCompleted() on line 156 bypasses it.
           If m_GameEnded has no subscribers, OnEventRaised is null
           and this line throws.

ISSUE 4: Double subscription in ScoreObjectiveSO tests
  Severity: LOW — test noise, not a failure
  Cause: ScriptableObject.CreateInstance auto-calls OnEnable (which throws
         because fields are null), then we call OnEnable again via reflection
         after setting fields. The first OnEnable is suppressed with
         LogAssert.ignoreFailingMessages. The second OnEnable also calls
         NullRefChecker.Validate, which may log warnings for non-critical
         fields. These logs do not cause test failures because
         ignoreFailingMessages is already reset by then and NullRefChecker
         uses LogError only for null required fields.
  Note: If subscription count matters, the first OnEnable's subscription
        attempt failed (NullRef), so only the second subscription is active.
